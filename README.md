# 日程調整ボード

PCのブラウザで `index.html` を開くと使えます。

## できること

- 9:00-18:00 の30分刻みで可否入力
- 複数の候補日を追加
- 開始日と終了日を指定して候補日を一括追加
- 複数の打ち合わせをタブで切り替え
- 可否欄をドラッグして連続入力
- Excelで文字化けしにくいCSVを書き出し
- 複数参加者の可否を1画面で集計
- 入力内容をブラウザ内に自動保存
- JSONでバックアップ/復元
- CSVで集計結果を書き出し

## 使い方

1. `index.html` をダブルクリックして開く
2. 打ち合わせ名を入力。複数ある場合は「打ち合わせを追加」でタブを増やす
3. 候補日を追加、または開始日と終了日を指定して期間をまとめて追加
4. 参加者を追加
5. 参加者ごとに「入力」を押して、時間枠をクリックまたはドラッグ
6. 「集計」タブで空いている人数を確認

JSONを書き出しておくと、別の日に読み込んで続きから編集できます。


## リアルタイム共有

Supabaseを使うと、同じ共有リンクを開いた人同士で入力内容が同期されます。
共有URLは `?board=...&token=...` 形式です。`board` だけでは共有モードにならず、`token` が一致しない会議は読み書きできません。

1. Supabaseで新しいプロジェクトを作る
2. `supabase-schema.sql` の内容を Supabase の SQL Editor で実行する
3. `config.js` に Project URL と anon public key を入れる
4. `index.html` を開き、「共有リンクを作成」を押す
5. 「リンクをコピー」で参加者に共有する

## セキュリティ

- `share_id` とは別に256bit相当の `access_token` を生成します。
- 読み書きはRPC経由で行い、`share_id` と `access_token` が両方一致した場合だけ成功します。
- テーブルへのanon直接SELECT/INSERT/UPDATEはRLSで拒否します。
- 他の会議一覧を取得するAPIはありません。
- 共有リンクを知っている人は閲覧・編集できます。リンク自体が秘密情報なので、公開場所には貼らないでください。

Realtime同期はPostgres changesではなく、`share_id + access_token` ごとのBroadcast通知を使います。通知には会議データを載せず、受信側が `access_token` 付きRPCで最新データを再取得します。Database Replicationで `schedule_boards` を公開する必要はありません。
